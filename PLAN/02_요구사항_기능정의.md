# 02. 요구사항 및 기능 정의

## 1) 핵심 기능 요구사항

### A. 출석 보너스

1. 연속 출석 보너스
   - 주간/월간/연간 보너스는 **연속일수 기준이 아니라 달력 기간 완주 기준**으로 판정
   - 주간: 해당 주(월~일 또는 운영정책에서 정의한 주 시작일 기준)의 모든 날짜 출석 시 지급
   - 월간: 해당 월 1일~말일 **전일 출석** 시 지급
   - 연간: 해당 연도 1월 1일~12월 31일 **전일 출석** 시 지급
   - 예시
     - 3월 15일~4월 14일 연속 출석: 월간 보너스 아님
     - 3월 1일~3월 31일 연속 출석: 월간 보너스 지급
2. 일일 1~3위 보너스
   - 서버 기준 일자 단위로 1등/2등/3등 판별
   - 동시 요청 충돌 시 트랜잭션으로 순위 고정

### A-1. 기간 보너스 중복 지급 규칙

- 같은 날 주간/월간/연간 조건을 동시에 만족할 수 있다.
- 이 경우 **기간이 가장 긴 보너스 1개만 지급**한다.
  - 연간 > 월간 > 주간 우선순위
  - 예: 주간+월간 동시 만족 시 월간만 지급

### B. 출석 UX

1. 출석 시 인삿말 입력
   - 사용자가 직접 입력 가능
   - 기본 인삿말 풀에서 랜덤 문구를 기본값으로 노출
2. 출석부 캘린더
   - 출석 날짜 표시
   - 결석 날짜 표시
3. 현재 시간 실시간 표시
   - 페이지 진입 후 초 단위 갱신
4. 상세보기 드롭다운
   - 클릭 시 설정/보너스 정책 요약 노출

### C. 설정 기능

1. 자동출석 사용 여부
   - 로그인/자동로그인 시 자동 출석 처리 여부
2. 출석 가능 시간 설정
   - 예: 01:00~23:00
   - 기본값: 시간 제한 없음(종일 가능)
3. 랜덤 포인트 사용 여부
   - 확률 기반 추가 포인트 지급
   - 최소/최대 범위 + 확률(%) 설정

## 2) 사용자 시나리오

### 시나리오 1: 일반 사용자 수동 출석

- 사용자가 출석 페이지 접속
- 랜덤 기본 인삿말 확인 후 수정 가능
- 출석 버튼 클릭
- 당일 출석/순위/연속일/획득 포인트 확인

### 시나리오 2: 자동출석 활성화 사용자

- 로그인 성공 직후 자동출석 조건 검사
- 가능 시간/중복 여부/권한 확인
- 출석 성공 시 사용자 알림(토스트 또는 배지)

### 시나리오 3: 관리자 정책 변경

- 관리자 페이지에서 출석 가능 시간 변경
- 저장 즉시 다음 요청부터 새 정책 적용
- 변경 이력 로그 기록(권장)

## 3) 예외 케이스

- 이미 출석한 사용자의 중복 요청
- 출석 가능 시간이 아닌 경우
- 탈퇴/정지/권한 제한 사용자
- 서버 시간 변경/타임존 이슈
- 일자 경계(23:59:59~00:00:01) 동시 요청
