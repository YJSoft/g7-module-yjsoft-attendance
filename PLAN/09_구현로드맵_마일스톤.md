# 09. 구현 로드맵 및 마일스톤

## Phase 0. 준비

- 레퍼런스 리포지토리 구조 확인
  - g7 코어
  - sirsoft-page/board/ecommerce 모듈
  - sirsoft-basic 템플릿
- 모듈 네이밍/라우팅/권한 패턴 수집
- 상세 정책 확정(월간 연속 기준, 자정 경계 규칙)

## Phase 1. 기초 골격

- 모듈 스캐폴딩
- module.json/서비스 프로바이더/라우트 등록
- 마이그레이션/모델/리포지토리 기본 구성

완료 기준:
- 모듈 활성화/비활성화 정상
- 기본 API 헬스체크 가능

## Phase 2. 핵심 출석 엔진

- 출석 체크 서비스 구현
- 중복 출석 방지(유니크+트랜잭션)
- 일일 순위 산정(1~3등)
- 연속 출석(streak) 계산 + 기간완주 보너스 판정(주/월/년)
- 기간 보너스 중복 시 장기 기간 1건만 지급(연간 > 월간 > 주간)
- 포인트 합산 및 로그

완료 기준:
- 단위 테스트로 주요 분기 100% 통과(핵심 로직 기준)

## Phase 3. 사용자 UI

- 출석부 레이아웃 구성
- 실시간 현재 시각 표시
- 캘린더 출석/결석 시각화
- 상세보기 드롭다운
- 인삿말 랜덤 기본값 연결

완료 기준:
- 사용자 시나리오 1~2 정상 동작

## Phase 4. 관리자 설정 UI/API

- 자동출석/시간설정/랜덤포인트 설정 화면
- 저장/조회 API 연동
- 유효성 검증 및 에러 처리

완료 기준:
- 설정 변경 즉시 정책 반영

## Phase 5. 안정화/릴리스

- 예외 처리/로깅 강화
- 성능 점검(월별 조회/동시 출석)
- 문서화/CHANGELOG 작성
- `v1.0.0` 태깅 준비

완료 기준:
- 릴리스 체크리스트 전 항목 완료
