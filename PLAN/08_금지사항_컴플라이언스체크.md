# 08. 금지사항 및 컴플라이언스 체크리스트 (AGENTS + 하위문서 통합)

## 1) API/핸들러 금지 패턴

- `G7Core.actions.execute` 금지 → `G7Core.dispatch` 사용
- `G7Core.api.call` 직접 호출 금지
- `handler: "api"` 금지 → `handler: "apiCall"`
- `handler: "nav"` 금지 → `handler: "navigate"`
- `handler: "setLocalState"` 금지 → `handler: "setState" + target: "local"`

## 2) 레이아웃/컴포넌트 금지 패턴

- HTML 태그 직접 사용 금지(`<div>`, `<button>` 등)
- Form 내 Button `type` 생략 금지
- `options={{options}}` 단독 사용 금지(기본값 필요)
- iteration에서 `item/index` 직접 변수명 사용 금지

## 3) 데이터 바인딩/표현식 금지 패턴

- fallback 없는 바인딩 금지 (`{{value}}` 단독 금지)
- 오류 객체 오기 접근 금지 (`{{error.data}}`)
- 이벤트 값 축약 `$value` 금지 (`$event.target.value` 사용)
- await 이후 오래된 상태 스냅샷 재사용 금지

## 4) 백엔드 계층 금지 패턴

- Controller에 검증 로직 구현 금지(FormRequest 사용)
- Controller에서 Repository 직접 주입 금지(Service만 주입)
- Service에서 구체 Repository 클래스 직접 타입힌트 금지(Interface 주입)
- 사용자 API에서 관리자 권한 데이터/기능 처리 금지

## 5) 모듈 구조/배포 금지 패턴

- 코어 파일 수정 금지(모듈 확장 포인트 사용)
- `resources/layouts/` 루트에 레이아웃 파일 직접 배치 금지
- 모듈 버전 변경 시 CHANGELOG 누락 금지
- 마이그레이션 down() 미구현 금지, 컬럼 comment 누락 금지

## 6) 본 프로젝트 특화 금지사항

- 사용자 페이지에서 관리자 API 호출 금지 (절대)
- 출석 중복 제어를 프론트엔드에만 의존 금지(서버 유니크/트랜잭션 필수)
- 출석 판정을 클라이언트 시간 기준으로 처리 금지(서버 기준)
- 주/월/년 보너스를 단순 연속일수(7/30/365)로 판정하는 구현 금지
- 주/월/년 동시 만족 시 중복 지급 금지(연간 > 월간 > 주간 1건만 지급)
