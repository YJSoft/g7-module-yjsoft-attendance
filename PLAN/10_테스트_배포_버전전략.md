# 10. 테스트, 배포, 버전 전략 (testing-guide / changelog-rules 반영)

## 1) 테스트 원칙

- 테스트 통과 = 작업 완료.
- 기존 테스트가 있으면 수정 반영 후 반드시 재실행.
- 기존 테스트가 없으면 신규 테스트 작성 후 실행.
- `_bundled` 모듈은 `_bundled` 경로에서 직접 테스트 실행 가능.

## 2) 백엔드 테스트 계획

### 단위 테스트

- AttendanceService
  - 중복 출석
  - 출석 가능 시간
  - 연속 출석 계산
  - 보너스 계산(주간/월간/연간/순위/랜덤)
  - 기간 보너스 중복 우선순위(연간 > 월간 > 주간)

### 통합 테스트

- checkin API 성공/실패 분기
- calendar 조회
- 관리자 설정 저장/조회
- 자동출석 트리거
- 기간 보너스 판정 케이스
  - 3/15~4/14 연속 출석: 월간 미지급
  - 3/1~3/31 출석: 월간 지급
  - 같은 날 주간+월간 동시 만족: 월간만 지급

### 예시 실행 커맨드

- `php vendor/bin/phpunit modules/_bundled/yjsoft-attendance/tests`
- `php vendor/bin/phpunit --filter=AttendanceCheckinControllerTest modules/_bundled/yjsoft-attendance/tests`

## 3) 프론트엔드 테스트 계획

- 레이아웃 JSON 렌더링 테스트
- 핸들러 동작 테스트(`apiCall`, `setState`)
- 바인딩 fallback/Optional Chaining 테스트
- 예시: `cd modules/_bundled/yjsoft-attendance && npm run test:run`

## 4) 데이터베이스/마이그레이션 검증

- 컬럼 comment(한국어) 포함 여부
- `down()` 구현 여부
- 유니크 인덱스(`user_id + attendance_date`) 검증
- 동시성 테스트(중복 출석 race condition)

## 5) 버전/CHANGELOG 전략

- 시작 버전: `1.0.0`
- 버전 규칙: SemVer
- 버전 상승 시 모듈 루트 `CHANGELOG.md` 필수 업데이트
  - Keep a Changelog 카테고리 사용
  - 날짜 형식: `YYYY-MM-DD`
