# 07. 관리자 설정 및 운영 정책 (module-settings 정렬)

## 1) 설정 저장 구조

- 기본 설정은 `config/settings/defaults.json`에 정의한다.
- 런타임 설정 저장은 모듈 격리 경로를 사용한다(코어 설정과 분리).
- 설정 조회는 `module_setting('yjsoft-attendance', 'key')` 패턴을 기준으로 사용한다.

## 2) 설정 항목

1. 자동출석
   - `auto_checkin_enabled: bool`
2. 출석 가능 시간
   - `checkin_time_limited: bool`
   - `checkin_start: HH:mm`
   - `checkin_end: HH:mm`
3. 랜덤 포인트
   - `random_bonus_enabled: bool`
   - `random_bonus_probability: 0~100`
   - `random_bonus_min/max`
4. 기본 인삿말 풀
   - `default_greetings: string[]`

## 3) 설정 API/Controller 원칙

- 관리자 설정 API는 `AdminBaseController` 계층에서만 제공.
- 설정 입력은 전용 FormRequest로 검증.
- 서비스 계층에서 정책 충돌(예: min > max) 재검증.

## 4) 프론트 노출 제어

- 사용자 페이지에 필요한 공개 설정만 노출한다.
- 민감/운영 내부 설정값은 frontend_schema(또는 동등한 정책)로 숨긴다.
- 관리자 설정 원본 전체를 사용자 API로 반환하지 않는다.

## 5) 운영 가이드

- 정책 변경 시 즉시 반영되도록 캐시 무효화/재로딩 정책을 포함한다.
- 설정 변경 이력(누가, 언제, 무엇)을 관리자 로그로 남긴다.
